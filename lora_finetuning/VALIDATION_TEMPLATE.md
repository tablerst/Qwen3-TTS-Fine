# LoRA 固定验证模板

用于配合 `lora_finetuning/sft_12hz_lora.py` 的 validation / best-checkpoint / early stopping 流程做统一试听与打分。

---

## 推荐固定验证集规模

建议总数控制在 **8~12 条**，保证每次 checkpoint 对比时输入完全一致。

### 日语（核心目标，建议 6 条）

1. 中性陈述句 × 2
2. 情绪句 × 2
3. 长句 × 1
4. 角色代表句 × 1

### 其他语言 sanity check（建议 2 条）

1. 中文 × 1
2. 英语 × 1

### instruct 维度

建议固定 2~4 个风格标签，例如：

- 平静
- 正式
- 温柔
- 激动

---

## 建议记录字段

每条验证样本建议至少包含：

- `id`
- `language`
- `text`
- `instruct`
- `notes`

如果后续单独维护 `validation_jsonl`，建议保证文本不频繁变动，避免跨实验结果不可比。

---

## 人工评分表

每条样本按 1~5 分记录以下维度：

| 维度 | 说明 | 分数 |
| --- | --- | --- |
| 发音正确性 / 可懂度 | 是否吞字、错音、断裂；中日英是否还能正常发声 |  |
| 音质稳定性 | 是否有杂音、电流音、破音、失真 |  |
| 人设相似度 / 风格感 | 是否更接近目标角色音色、气质、韵律 |  |
| 指令跟随度 | instruct 是否有效，不同情绪是否可区分 |  |

### 推荐权重

- 发音正确性：0.35
- 音质稳定性：0.35
- 人设相似度：0.20
- 指令跟随度：0.10

---

## early stopping 参考规则

### 硬停

出现以下任一情况，建议直接停止继续训练：

- 日语核心样本出现明显杂音 / 电流音
- 语音无法正常发词
- 相比上一 checkpoint 音质明显恶化
- 中文和英语从可用跌到基本不可用，同时日语收益也不明显

### patience 停止

若连续 2 次验证满足：

- 总分没有超过历史最佳
- 人设感没有继续增强
- 泛化却在下降

建议提前停训并回退到最佳 checkpoint。

---

## 推荐归档方式

每个实验目录建议保留：

- `validation_records.json`：训练脚本实际使用的 validation 样本
- `metrics.json`：epoch loss / validation loss / best-checkpoint 信息
- 人工试听记录表：可单独保存为 markdown 或 spreadsheet

这样可以把“自动指标”和“人工听感”串起来，避免只看 loss 选错 checkpoint。
